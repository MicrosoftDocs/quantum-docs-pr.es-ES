## <a name="classifier-training-as-a-supervised-learning-task"></a><span data-ttu-id="30a78-137">Aprendizaje clasificador como una tarea de aprendizaje supervisado</span><span class="sxs-lookup"><span data-stu-id="30a78-137">Classifier training as a supervised learning task</span></span>

<span data-ttu-id="30a78-139">Aquí nos referimos a nosotros solo con la clasificación de dos niveles, es decir, el caso de $d = $2 y solo dos clases con las etiquetas $y _ 1, y_2 $.</span><span class="sxs-lookup"><span data-stu-id="30a78-139">Here, we concern ourselves with two level classification only, i.e. the case of $d=2$ and only two classes with the labels $y_1,y_2$.</span></span>

### <a name="likelihood-as-the-training-goal"></a><span data-ttu-id="30a78-141">Probabilidad como objetivo de entrenamiento</span><span class="sxs-lookup"><span data-stu-id="30a78-141">Likelihood as the training goal</span></span>

<span data-ttu-id="30a78-142">Dado un circuito de Quantum más aprendido $U (\theta) $, donde $ \theta $ es un vector de parámetros y que denota la medida final de $M $, la probabilidad media de la inferencia de etiqueta correcta es $ $ \begin{align} \mathcal{L} (\theta) = \frac {1} {| \mathcal{D} |} \left (\ sum_ {(x, y_1) \In\mathcal{D}} P (M = Y_1 | U (\theta) x) + \ sum_ {(x y_2) \in\mathcal{D}} P (M = y_2 | U (\theta) x) \right) \end{align} $ $ Where $P (M = y | z) $ es la probabilidad de medir $y $ en el estado de Quantum $z $.</span><span class="sxs-lookup"><span data-stu-id="30a78-142">Given a learnable quantum circuit $U(\theta)$, where $\theta$ is a vector of parameters, and denoting the final measurement by $M$, the average likelihood of the correct label inference is $$ \begin{align} \mathcal{L}(\theta)=\frac{1}{|\mathcal{D}|} \left( \sum_{(x,y_1)\in\mathcal{D}} P(M=y_1|U(\theta) x) + \sum_{(x,y_2)\in\mathcal{D}} P(M=y_2|U(\theta) x)\right) \end{align} $$ where $P(M=y|z)$ is the probability of measuring $y$ in quantum state $z$.</span></span>
<span data-ttu-id="30a78-143">En este caso, basta con comprender que la función de probabilidad $ \mathcal{L} (\theta) $ es fluida en $ \theta $ y su derivado en cualquier $ \ theta_j $ se puede calcular esencialmente en el mismo protocolo Quantum que se usa para calcular la función de probabilidad.</span><span class="sxs-lookup"><span data-stu-id="30a78-143">Here, it suffices to understand that the likelihood function $\mathcal{L}(\theta)$ is smooth in $\theta$ and its derivative in any $\theta_j$ can be computed by essentially the same quantum protocol as used for computing the likelihood function itself.</span></span> <span data-ttu-id="30a78-144">Esto permite optimizar $ \mathcal{L} (\theta) $ por descenso de gradiente.</span><span class="sxs-lookup"><span data-stu-id="30a78-144">This allows for optimizing the $\mathcal{L}(\theta)$ by gradient descent.</span></span>

### <a name="classifier-bias-and-training-score"></a><span data-ttu-id="30a78-145">Puntuación del clasificador y puntuación de entrenamiento</span><span class="sxs-lookup"><span data-stu-id="30a78-145">Classifier bias and training score</span></span>

<span data-ttu-id="30a78-146">Dados algunos valores intermedios (o finales) de los parámetros en $ \theta $, es necesario identificar un único valor real $b $ know como clasificación del *clasificador* para realizar la inferencia.</span><span class="sxs-lookup"><span data-stu-id="30a78-146">Given some intermediate (or final) values of the parameters in $\theta$, we need to identify a single real value $b$ know as *classifier bias* to do the inference.</span></span> <span data-ttu-id="30a78-147">La regla de inferencia de etiqueta funciona de la siguiente manera:</span><span class="sxs-lookup"><span data-stu-id="30a78-147">The label inference rule works as follows:</span></span> 
- <span data-ttu-id="30a78-148">En un ejemplo $x $ se le asigna la etiqueta $y _2 $ si y solo si $P (M = y_2 | U (\theta) x) + b > $0,5 (RULE1) (de lo contrario, se le asigna etiqueta $y _ 1 $)</span><span class="sxs-lookup"><span data-stu-id="30a78-148">A sample $x$ is assigned label $y_2$ if and only if $P(M=y_2|U(\theta) x) + b > 0.5$  (RULE1) (otherwise it is assigned label $y_1$)</span></span>

<span data-ttu-id="30a78-149">Claramente $b $ debe estar en el intervalo $ (-0.5, + 0.5) $ para que sea significativo.</span><span class="sxs-lookup"><span data-stu-id="30a78-149">Clearly $b$ must be in the interval $(-0.5,+0.5)$ to be meaningful.</span></span>

<span data-ttu-id="30a78-150">Un caso de entrenamiento $ (x, y) \en \mathcal{D} $ se considera una *clasificación* incorrecta dada la diferencia $b $ si la etiqueta deducida para $x $ como por RULE1 es realmente diferente de $y $.</span><span class="sxs-lookup"><span data-stu-id="30a78-150">A training case $(x,y) \in \mathcal{D}$ is considered a *misclassification* given the bias $b$ if the label inferred for $x$ as per RULE1 is actually different from $y$.</span></span> <span data-ttu-id="30a78-151">El número total de clasificaciones incorrectas es la *puntuación de entrenamiento* del clasificador dada la diferencia $b $.</span><span class="sxs-lookup"><span data-stu-id="30a78-151">The overall number of misclassifications is the *training score* of the classifier given the bias $b$.</span></span> <span data-ttu-id="30a78-152">La diferencia de clasificador *óptima* $b $ minimiza la puntuación de entrenamiento.</span><span class="sxs-lookup"><span data-stu-id="30a78-152">The *optimal* classifier bias $b$ minimizes the training score.</span></span> <span data-ttu-id="30a78-153">Es fácil ver eso, según las estimaciones de probabilidad precalculadas $ \{ P (M = y_2 | U (\theta) x) | (x, \*) \in\mathcal{D} \} $, la diferencia de clasificador óptima se puede encontrar mediante la búsqueda binaria en el intervalo $ (-0.5, + 0.5) $ mediante la realización de un máximo de $ \ log_2 (| \mathcal{D} |) $ Steps.</span><span class="sxs-lookup"><span data-stu-id="30a78-153">It is easy to see that, given the precomputed probability estimates $\{ P(M=y_2|U(\theta) x) | (x,\*)\in\mathcal{D} \}$, the optimal classifier bias can be found by binary search in interval $(-0.5,+0.5)$ by making at most $\log_2(|\mathcal{D}|)$ steps.</span></span>
