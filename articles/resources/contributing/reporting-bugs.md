# <a name="reporting-bugs"></a>Informes de errores #

Si cree que ha encontrado un error en un componente del kit de desarrollo de Quantum, nos agradecería mucho un informe.
Después de todo, es posible que otra persona esté lidiando con el mismo problema. nos permite saber cómo solucionarlo para todos.
El primer paso para informar de un error es comenzar por los problemas existentes encontrados en cada repositorio, ya que es muy posible que otra persona haya notificado el mismo problema.

Si el problema ya se ha encontrado antes, puede resultar útil proporcionar información adicional o contexto que nos ayude a diagnosticar y resolver el problema.
En ese caso, no dude en participar en los debates sobre el problema.

Si el error no se ha comunicado antes, le agradecemos que abra un nuevo problema en el repositorio del componente en cuestión.
Al crear un nuevo problema, se le solicitará una plantilla que sugiera información que a menudo resulta útil para crear informes de errores informativos:

- Pasos para reproducir el problema
- Lo que esperaba que ocurriera
- Qué sucedió realmente
- Información sobre el entorno de software (por ejemplo, versiones del kit de desarrollo para el sistema operativo, .NET Core y Quantum)

También puede [crear una solicitud de incorporación](https://help.github.com/articles/about-pull-requests/) de cambios para corregir el error directamente, si es muy sencillo y no merece la pena la explicación (por ejemplo, un error tipográfico).
