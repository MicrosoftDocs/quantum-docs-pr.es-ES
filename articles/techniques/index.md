<span data-ttu-id="12f16-105">En esta sección de la documentación se ofrecen detalles de los conceptos más importantes usados para crear programas cuánticos en Q#, así como para interactuar con ellos desde aplicaciones clásicas.</span><span class="sxs-lookup"><span data-stu-id="12f16-105">This section of our documentation details the core concepts used to create quantum programs in Q#, and to interact with those programs from classical applications.</span></span>

<span data-ttu-id="12f16-107">Su contenido es el siguiente.</span><span class="sxs-lookup"><span data-stu-id="12f16-107">Their contents are as follows.</span></span>

- <span data-ttu-id="12f16-108">En [Introducción a los programas en Q#](xref:microsoft.quantum.techniques.file-structure) se proporciona información general sobre el propósito y la funcionalidad del lenguaje de programación Q#.</span><span class="sxs-lookup"><span data-stu-id="12f16-108">[Q# program overview](xref:microsoft.quantum.techniques.file-structure) provides an overview of the purpose and functionality of the Q# programming language.</span></span> 
    <span data-ttu-id="12f16-109">En concreto, se aclara que Q# *no* es un lenguaje que simplemente sirve para simular la mecánica cuántica (aunque indudablemente nuestro simulador de estado completo ofrece esa funcionalidad).</span><span class="sxs-lookup"><span data-stu-id="12f16-109">In particular, it clarifies how Q# is *not* a language for merely simulating quantum mechanics---though that functionality is of course provided by our full state simulator.</span></span> 
    <span data-ttu-id="12f16-110">En su lugar, Q# se diseñó pensando en el futuro y sus programas describen la forma en que un equipo de control clásico *interactúa* con qubits.</span><span class="sxs-lookup"><span data-stu-id="12f16-110">Rather, Q# was designed with an eye on the future, and its programs describe how a classical control computer *interacts* with qubits.</span></span> 

- <span data-ttu-id="12f16-111">En [Operaciones y funciones](xref:microsoft.quantum.techniques.opsandfunctions) se proporcionan detalles de los dos tipos a los que se puede llamar del lenguaje Q#: *operaciones*, que incluyen la acción en sistemas de qubits y cuántico; y *funciones*, que funcionan estrictamente con información clásica.</span><span class="sxs-lookup"><span data-stu-id="12f16-111">[Operations and functions](xref:microsoft.quantum.techniques.opsandfunctions) details the two callable types of the Q# language: *operations*, which include action on qubits and quantum systems; and *functions*, which strictly work with classical information.</span></span> 
    <span data-ttu-id="12f16-112">Si la información clásica y la cuántica no funcionaran en tándem, sería imposible acceder a la informática cuántica.</span><span class="sxs-lookup"><span data-stu-id="12f16-112">Without both classical and quantum information working in tandem, quantum computing would remain out of reach.</span></span> 
    <span data-ttu-id="12f16-113">En esta sección se describe cómo definir y usar estos tipos a los que se puede llamar en el flujo de control de un programa en Q#.</span><span class="sxs-lookup"><span data-stu-id="12f16-113">This section describes how to define and use these callables within the control flow of a Q# program.</span></span>

- <span data-ttu-id="12f16-114">En [Variables locales](xref:microsoft.quantum.techniques.local-variables) se describe el rol de las variables dentro de los programas en Q# y cómo sacarles provecho de forma eficaz.</span><span class="sxs-lookup"><span data-stu-id="12f16-114">[Local variables](xref:microsoft.quantum.techniques.local-variables) describes the role of variables within Q# programs and how to leverage them effectively.</span></span> 
    <span data-ttu-id="12f16-115">En concreto, aprenderá no solo en qué se diferencian las variables inmutables y las mutables, sino también cómo asignarlas y reasignarlas.</span><span class="sxs-lookup"><span data-stu-id="12f16-115">In particular, you will learn the difference between immutable/mutable variables and how to assign/re-assign them.</span></span>

- <span data-ttu-id="12f16-116">En [Uso de qubits](xref:microsoft.quantum.techniques.qubits) se describen las características de Q# que puede usar para trabajar tanto con qubits individuales como con sistemas de qubits.</span><span class="sxs-lookup"><span data-stu-id="12f16-116">[Working with qubits](xref:microsoft.quantum.techniques.qubits) describes the features of Q# that you can use to address individual qubits and systems of qubits.</span></span> 
    <span data-ttu-id="12f16-117">En concreto, eso conlleva su asignación, la realización de operaciones en ellos y, en último término, su medición.</span><span class="sxs-lookup"><span data-stu-id="12f16-117">Specifically, that entails their allocation, performing operations on them, and ultimately their measurement.</span></span> 
    <span data-ttu-id="12f16-118">Además, aprenderá algunas técnicas útiles de flujo de control.</span><span class="sxs-lookup"><span data-stu-id="12f16-118">Additionally, you will learn some useful control flow techniques.</span></span>

- <span data-ttu-id="12f16-119">En [Reunir todo](xref:microsoft.quantum.techniques.puttingittogether), sacará provecho de las técnicas de las secciones anteriores para crear un programa que realice **teleportabilidad cuántica**: usar dos bits clásicos para "teletransportar" el estado completo de un qubit a otro.</span><span class="sxs-lookup"><span data-stu-id="12f16-119">In [Putting it all together](xref:microsoft.quantum.techniques.puttingittogether), you will leverage the techniques from the sections above to create a program which performs **quantum teleportation**: using two classical bits to "teleport" the full state of one qubit onto another.</span></span>

- <span data-ttu-id="12f16-120">En [Ir más allá](xref:microsoft.quantum.techniques.going-further) se presentan técnicas avanzadas que pueden resultar útiles a la hora de pasar a una programación cuántica más compleja.</span><span class="sxs-lookup"><span data-stu-id="12f16-120">[Going further](xref:microsoft.quantum.techniques.going-further) introduces advanced techniques that can prove helpful as you move toward more complex quantum programming.</span></span> 
    <span data-ttu-id="12f16-121">En concreto, se explica el uso de operaciones y funciones *con parámetros de tipo* en Q #, lo que permiten un flujo de control de orden superior independiente de los tipos concretos de su entrada y salida, así como *pedir prestados* qubits.</span><span class="sxs-lookup"><span data-stu-id="12f16-121">In particular, we discuss the use of *type-parameterized* operations and functions in Q#, which enable higher-order control flow by remaining agnostic to the specific types of their input/output, as well as *borrowing* qubits.</span></span> 
    <span data-ttu-id="12f16-122">La diferencia entre esto último y la asignación básica de qubits es que las operaciones de Q# pueden usar qubits "sucios" (que no se han inicializado necesariamente en un estado conocido) como ayuda para los cálculos.</span><span class="sxs-lookup"><span data-stu-id="12f16-122">The latter differs from basic qubit allocation in that a Q# operation may use "dirty" qubits---qubits not necessarily initialized to a known state---to assist computations.</span></span>

- <span data-ttu-id="12f16-123">En [Pruebas y depuración](xref:microsoft.quantum.techniques.testing-and-debugging), se muestran con todo lujo de detalles, algunas técnicas para asegurarse de que el código funciona como debería.</span><span class="sxs-lookup"><span data-stu-id="12f16-123">[Testing and debugging](xref:microsoft.quantum.techniques.testing-and-debugging) details some techniques for making sure your code is doing what it is supposed to do.</span></span> 
    <span data-ttu-id="12f16-124">Dada la opacidad general de la información acerca de lo cuántico, la depuración de un programa cuántico puede requerir técnicas especializadas.</span><span class="sxs-lookup"><span data-stu-id="12f16-124">Due to the general opacity of quantum information, debugging a quantum program can require specialized techniques.</span></span> 
    <span data-ttu-id="12f16-125">Afortunadamente, Q# admite muchas de las técnicas de depuración clásicas a las que están acostumbrados los programadores, así como otras que son específicas del mundo cuántico.</span><span class="sxs-lookup"><span data-stu-id="12f16-125">Fortunately, Q# supports many of the classical debugging techniques programmers are used to, as well as those that are quantum-specific.</span></span> <span data-ttu-id="12f16-126">Entre ellas se incluye la creación y ejecución de pruebas unitarias en Q#, la inserción de *aserciones* en los valores y probabilidades del código y las funciones `Dump` cuyo resultado es el estado de la máquina de destino.</span><span class="sxs-lookup"><span data-stu-id="12f16-126">These include creating/running unit tests in Q#, embedding *assertions* on values and probabilities in your code, and the `Dump` functions which output the state of target machine.</span></span> 
    <span data-ttu-id="12f16-127">Estas últimas se pueden usar junto con el simulador de estado completo para depurar ciertas partes de los cálculos, ya que se sortean algunas de las limitaciones del entorno cuántico (por ejemplo, el teorema de no clonación).</span><span class="sxs-lookup"><span data-stu-id="12f16-127">The latter can be used alongside our full state simulator to debug certain parts of computations by skirting some quantum limitations (e.g. the no-cloning theorem).</span></span>
